# Shell Parameters

In shell scripting, parameters are entities that store values and play a crucial role in the execution of commands and scripts. Understanding the various types of parameters is essential for efficient shell scripting.

## What is a Parameter?

A parameter, in the context of shell scripting, refers to any entity that can store values. These values can be manipulated and used within the script to control its behavior.

## Types of Parameters

There are three primary types of parameters in shell scripting:

### 1. Variables

Variables are perhaps the most common type of parameter used in shell scripting. They serve as placeholders for storing data, which can be manipulated and accessed throughout the script's execution. There are two main types of variables:

#### a. User-defined Variables

User-defined variables are created by the script developer to store custom data. These variables are assigned values using the assignment operator (`=`) and can hold various types of information, such as strings, integers, or arrays.

Example:

```bash
name="John"
age=30
```

#### b. Shell Variables

Shell variables are predefined by the shell environment and often used to control various aspects of the shell's behavior. These variables are set by the shell or by shell scripts and can influence how commands are executed or how the shell interacts with the user.

Example:

```bash
echo "The current shell is: $SHELL"
echo "My username is: $USER"
```

### 2. Positional Parameters

Positional parameters refer to the arguments or values supplied to a shell script or function when it is executed. These parameters are accessed within the script using special notation (`$1`, `$2`, `$3`, etc.), where `$1` represents the first parameter, `$2` the second parameter, and so on. Additionally, `$0` represents the name of the script or function itself.

Example:

```bash
#!/bin/bash

echo "The name of the script is: $0"
echo "The first parameter is: $1"
echo "The second parameter is: $2"
```

### 3. Special Parameters

Special parameters are predefined by the shell and hold specific meanings or values. They provide information about the script's execution environment, status, and other relevant details. Some commonly used special parameters include:

- `$$`: The process ID (PID) of the current shell.
- `$?`: The exit status of the last command.
- `$#`: The number of positional parameters passed to the script.
- `$@`: Represents all the positional parameters passed to the script as separate quoted strings.
- `$*`: Represents all the positional parameters passed to the script as a single quoted string.

Example:

```bash
#!/bin/bash

echo "Process ID: $$"
echo "Exit status of last command: $?"
echo "Number of parameters: $#"
echo "All parameters as separate strings: $@"
echo "All parameters as a single string: $*"
```
